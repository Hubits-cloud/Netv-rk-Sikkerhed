SSH Setup — Required Steps for Exam
  • Set hostname (not "Router")
  • Set ip domain-name
  • crypto key generate rsa modulus 2048
  • Set ip ssh version 2
  • Configure login local + transport input ssh on VTY
Exam question: "Which three additional steps are required?" → hostname, domain-name, crypto key generate (NOT DNS, NOT pre-shared keys)
login block-for — Read This Carefully
login block-for 150 attempts 4 within 90
  • Block for 150 seconds (first number)
  • If 4 attempts fail (middle number)
  • Within 90 seconds (last number)
Exam trap: People confuse the numbers. Remember: block-for = duration, within = window. The block-for value ALWAYS wins.
access control lists
Standard vs Extended ACLs
StandardExtended
Numbers1–99, 1300–1999100–199, 2000–2699
MatchesSource IP onlySrc + Dst + Protocol + Port
Place nearDestinationSource
IOS masksWildcard masks (0=match, 1=ignore)
ASA masksSubnet masks (NOT wildcard!)
Implicit deny: Every ACL ends with invisible deny any any. Unmatched traffic is dropped.
Router-generated packets cannot be filtered by an outbound ACL — they bypass it.
ACL Wildcard Masks
WildcardMeaningShortcut
0.0.0.0Match exact host (/32)host x.x.x.x
0.0.0.255Match /24 network—
0.0.255.255Match /16 network—
255.255.255.255Match anythingany
Wildcard = 255.255.255.255 − subnet mask
e.g. /24 mask = 255.255.255.0 → wildcard = 0.0.0.255
IPv6 ACLs: Named only. Must explicitly permit icmp any any nd-na and nd-ns or IPv6 breaks. Applied with ipv6 traffic-filter (not access-group).
ACL — Exam Traps
  • To change ACL direction: Remove inbound association, reapply outbound — do NOT delete and recreate
  • established keyword: Allows return TCP traffic (has ACK/RST bit set)
  • access-list 100 deny icmp 192.168.10.0 0.0.0.255 any echo reply applied outbound → NO traffic outbound (implicit deny catches everything else)
  • VTY restriction: Use access-class (not access-group) for lines
  • ASA standard ACLs identify destination IP (opposite of IOS!). Only used for OSPF routes — NOT applied to interfaces

ASA Key Facts
InterfaceLevelDefault Traffic Rule
inside100→ lower levels: permitted
dmz50→ inside: denied; → outside: permitted
outside0→ any higher: denied (need ACL)
  • To allow outside→inside: must configure an ACL
  • ASA ACLs use subnet masks not wildcard masks
  • Transparent mode = "bump in the wire"; invisible; no VPN/QoS/DHCP Relay
  • ASA show works in any mode (no do needed)
  • Object NAT: configured inside network objects; evaluated second
  • Twice NAT: standalone; evaluated first; more flexible
ZPF — Zone-Based Policy Firewall
self zone = system-defined; applies to traffic to/from the router itself
ActionEffect
inspectStateful — return traffic auto-allowed ✅
passOne-way permit; no state tracking
dropBlock (default for unmatched)
Config order: zones → assign interfaces → class-maps → policy-maps → zone-pairs → service-policy
ZPF benefits vs Classic: Not dependent on ACLs. Policies easy to read/troubleshoot. Default deny between zones. Interfaces can only be in one zone.
Traffic between same zone: permitted. No interface in a zone: cannot communicate with zoned interfaces.

Site-to-site VPN key type: symmetric (pre-shared or derived via DH). Exam answer: symmetric key.
Security lies in key secrecy, not the algorithm (Kerckhoffs's principle).
Hashing & Integrity
AlgorithmOutputStatus
MD5128-bitDeprecated
SHA-1160-bitDeprecated
SHA-256256-bitCurrent
SHA-512512-bitStrong
  • HMAC = hash + secret key → integrity and authentication (not confidentiality)
  • Digital signature = hash encrypted with private key → integrity + auth + non-repudiation
  • Salt = random value appended to password before hashing → defeats rainbow tables
Exam: MD5 and SHA = data integrity in IPsec. AES = confidentiality. DH = key exchange. RSA = authentication.

NTP, SNMP, Monitoring
  • NTP Stratum 0 = atomic/GPS clock. Each hop adds 1 stratum. Higher = less accurate.
  • Stratum 2 device = synced from stratum 1; can provide NTP to others
  • SNMPv3 = only version with auth + encryption. v1/v2c = community string in cleartext
  • SNMP TRAP = unsolicited alert from agent. INFORM = trap with acknowledgement
  • SIEM = centralised log correlation, alerting, forensic analysis
  • NetFlow = collects IP flow metadata (not payload). Used for anomaly detection
  • OOB management limitation on large networks: all devices appear on a single management network

Classic Exam Traps — Know These Cold
  • ASA standard ACL identifies destination (not source like IOS). Only used for OSPF routes.
  • ASA uses subnet masks in ACLs, NOT wildcard masks
  • ZPF self zone = system-defined; no interface assignment needed
  • single-connection keyword in TACACS+ config = keeps ONE TCP connection open, enhancing performance
  • IDS uses copies of traffic (SPAN/TAP); IPS sits inline with live traffic
  • Stateful firewall CANNOT stop application layer attacks
  • Router-generated packets bypass outbound ACLs
  • RADIUS function: provides separate ports for authorization and accounting — NOT separate AAA services (that's TACACS+)
  • Worm vs Virus: virus attaches to file + needs user action; worm self-replicates independently
  • SSL Appliance needed to decrypt SSL traffic so IPS can inspect it
  • AUP violation: First action = create firewall rule, OR revise AUP and get signatures (not suspend immediately)
  • OSPF MD5 auth does NOT encrypt routing updates — only authenticates them
